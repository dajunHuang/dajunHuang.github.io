一、低功耗远距离无线传输技术有哪些？并对其结构、特点和技术标准进行介绍

1、LoRa

LoRa 是指 Long Range Radio，它是专为远程低功耗通信设计的无线协议，主要针对M2M和物联网网络，由Semtech开发，能让公共或多租户网络能够连接到同一网络上运行的多个应用程序。

LoRa 为物联网应用提供远程、低功耗和安全的数据传输等功能。它能覆盖了比蜂窝网络更大的范围，可用于公共、专用或混合网络。它可以轻松嵌入现有基础设施，并支持使用低成本电池供电的物联网应用。

LoRa 的应用包括智能计量、库存跟踪、监控、汽车行业和公用事业等，它被广泛部署并与许多系统结合，甚至像 Arduino 这样的小型计算机也可以选择使用 LoRa，而且为大规模制造或专业的应用开发 LoRa 应用程序也很容易。

2. SigFox

SigFox 的工作频率为868或915 MHz，使用二进制相移键控（BPSK）传输非常少量的数据。它的传输速度非常慢（300 b/s）但可以实现远程覆盖，因此非常适合任何只需要少量数据的物联网应用。

SigFox 适用于任何需要发送少量、不频繁数据突发的应用程序。诸如基本报警系统、位置监控和简单计量等单向系统。SigFox的信号会重复发送以确保接收方能接收到，但信息的发送也会受到一些条件的限制，如供电电池寿命较短，缺乏确保信号塔接收到信息的能力。

3. LTE-M

LTE-M 是由 3GPP 发行版开发的低功耗广域网（LPWAN）无线电技术标准。LTE-M利用低成本模块，延长电池寿命，提供更好的信号穿透能力，并能够使用现有基础设施。

Cat M1在半双工模式下的上行链路和下行链路速度为375 kb/s，能够支持中低数据速率需求的物联网应用。在相同的速度下，LTE Cat M1可以通过无线（FOTA）提供远程固件更新。

与常规LTE的20 MHz相比，LTE-M 具有1.4 MHz的窄带宽，提供了更大的范围。它使用与常规 LTE 中相同的区域切换功能，可以使用LTE-M漫游，因此它适用于可以跨多个区域操作的应用程序。LTE-M 延迟在毫秒范围内，能为时间敏感型应用程序提供实时通信。即使终端设备未直接连接到电网，单次充电寿命也可达10年，维护成本低。

智能建筑、智能城市、汽车和交通、互联健康、智能计量是 LTE-M 应用的主要领域。

二、查阅资料，简述应用最广的物联网平台及其它们的结构、特点和应用领域

1、Thinworx8

Thinworx8 是工业公司的顶级物联网平台之一，它为设备提供了十分简单的连接，是一个提供构建，部署和扩展工业级别项目和应用的，更好，更快，更容易的平台。

ThingWorx 8于2017年6月8日上市， 拥有优越的平台能力，提供针对工程和制造领域而设计的角色应用、庞大的合作伙伴和客户生态系统、丰富的教学课程以及一系列专为工业物联网客户和合作伙伴设计的全新服务。

Thingworx是由PTC为开发企业应用程序开发而设计的物联网平台。它具有如下特性：

  • 能轻松连接到传感器与RFID等电子设备；

  • 一旦设置完毕就可以远程工作了；

  • 在dashboard预构建了小部件；

  • 移除了项目的复杂性；

  • 集成了机器学习；

2、Google Cloud IoT

Google Cloud IoT是专为智能物联网服务打造的平台，有一套完整的工具，可连接、处理、存储和分析边缘和云端的数据。该平台包含多种可伸缩的全托管式云服务，以及具备机器学习功能、用于边缘/本地计算的集成式软件堆栈。

Google云物联网平台提供了以下特性，包括：

  • 提供了巨大的存储；

  • 降低服务器维护成本；

  • 通过完全受保护，智能且响应迅速的物联网数据进行业务；

  • 高效且可扩展；

  • 分析大数据；

优点：

  • 最快的输入/输出；

  • 更短的访问时间；

  • 提供与其它google服务的集成；

缺点：

  • 大多数组件都是google的技术；

  • 编程语言会有限制；

物联网 (IoT) 是跨边缘和云的托管和平台服务的集合，用于连接、监视和控制数十亿项 IoT 资产。它还包括适用于装置和设备的安全性和操作系统，以及帮助企业构建、部署和管理 IoT 应用程序的数据和分析。

Azure IoT Suite提供了以下功能：

  • 简单的设备注册表；

  • 与SAP，Salesforce，Oracle，WebSphere等的丰富集成；

  • dashborad和可视化；

  • 实时流媒体；

优点：

  • 提供了第三方服务；

  • 安全且可扩展；

  • 高可用；

缺点：

  • 需要管理；

四、什么是Zigbee，Zigbee有什么特点

ZigBee协议是由ZigBee联盟制定的无线通信标准，该联盟成立于2001年8月。2002年下半年，英国Invensys公司、日本三菱电气公司、美国摩托罗拉公司以及荷兰飞利浦半导体公司共同宣布加入ZigBee联盟，研发名为“ZigBee”的下一代无线通信标准，这一事件成为该技术发展过程中的里程碑。ZigBee联盟现有的理事公司包括BM Group，Ember公司，飞思卡尔半导体，Honeywell，三菱电机，摩托罗拉，飞利浦，三星电子，西门子，及德州仪器。

ZigBee联盟的目的是为了在全球统一标准上实现简单可靠、价格低廉、功耗低、无线连接的监测和控制产品进行合作，并于2004年12月发布了第一个正式标准。

七、Zigbee无线网络分为哪几层，以及它与IEEE802.15.4 802.15.4的联系

完整的ZigBee 协议栈自上而下由应用层、应用汇聚层、网络层、数据链路层和物理层组成。

以下为各层在Zigbee结构中的作用：
物理层：作为Zigbee协议结构的最低层，提供了最基础的服务，为上一层MAC层提供了服务，如数据的接口等等。同时也起到了与现实 (物理) 世界交互的作用；
MAC层：负责不同设备之间无线数据链路的建立，维护，结束，确认的数据传送和接收；
网络/安全层：保证了数据的传输和完整性，同时可对数据进行加密；
应用/支持层：根据设计目的和需求使多个器件之间进行通信；
其中，Zigbee协议架构最具特色的两项是：功耗以及自组网。

zigbee和802.15.4都是无线通信协议。ZigBee建立在802.15.4标准之上，zigbee是以802.15.4为基础的，802.15.4只规定了通信协议栈的物理层和链路层的通信标准，zigbee将其扩展到整个通信协议，比如上层的网路层、应用层等，zigbee都规定了相应的通信标准。它确定了可以在不同制造商之间共享的应用纲要。
