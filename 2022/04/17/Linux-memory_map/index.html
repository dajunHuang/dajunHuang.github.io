Linux 内存分配和地址映射实验

发表于2022-04-17|更新于2022-04-17
|阅读量:

一、试验目的

  1. 掌握字符设备驱动程序中利用vm fault进行内存映射的方法。
  2. 掌握利用get_free_pages进行连续物理地址空间申请的方法。

二、实验环境

Linux ubuntu 3.2.14 #1 SMP Thu Feb 27 09:35:13 CST 2020 x86_64 x86_64 x86_64 GNU/Linux

三、实验内容及原理

实验内容

写一个简单的字符设备驱动程序，要求：

  • 在加载驱动程序时利用get_free_pages函数申请一片64KB的连续物理地址空间；

  • 利用vma fault机制实现对申请到的64KB地址空间进行内存映射；

  • 编写应用程序利用mmap进行内存映射，读写映射内存区域，通过打印输出观察具体每个页面实际进行内存映射的时机；

  • 在卸载驱动程序时利用free_pages释放申请到的64KB空间

mmp系统调用

mmap方法是file_operations结构的一部分，并且执行mmap系统调用时将调用该方法，实现物理地址空间和虚拟地址空间间的内存映射。

使用fault函数映射内存

当用户访问VMA中的页，而该页又不在内存中时，将调用相关的fault函数实现内存映射

运行结果

依次执行以下命令：

  1. mknod /dev/demo_drv c 98 0插入设备节点；
  2. make编译模块；
  3. sudo insmod demo_drv.ko插入模块；
  4. gcc -o app1 app1.c编译测试程序；
  5. ./app1执行测试程序查看控制台显示内容；
  6. dmesg查看缺页错误发生情况；
  7. sudo rmmod demo_drv卸载驱动。

执行测试程序后控制台显示：

dmesg显示：

测试程序中总共打印3 * 16行内容，前16行只打印地址，未产生缺页错误，之后16行访问了每页首地址的内容，此时共发生16次缺页错误，从打印的内核信息就能看到发生缺页错误的地址。第三遍访问时由于内核已经分配好的了内存，这时再访问已经分配好内存的页就不会再发生缺页错误。
