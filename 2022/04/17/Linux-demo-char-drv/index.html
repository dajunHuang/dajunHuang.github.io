Linux简单字符设备驱动

发表于2022-04-17|更新于2022-04-17
|阅读量:

一、试验目的

  1. 掌握简单字符设备驱动程序编写方法。
  2. 编写应用程序对驱动程序进行测试，学习应用程序与驱动程序之间的调用过程。

二、实验环境

Linux version 5.13.0-37-generic (buildd@lcy02-amd64-111) (gcc (Ubuntu 9.4.0-1ubuntu1~20.04) 9.4.0, GNU ld (GNU Binutils for Ubuntu) 2.34)

三、实验内容及原理

实验内容

写一个简单的字符设备驱动程序，要求:

  • 定义一个数据结构，该数据结构中包含一个大小为1024的buffer和一个count整形变量；定义一个全局的该数据结构指针，初始化为NULL；
  • 在open中对该指针进行NULL判断，为NULL则为其分配内存，并将数据结构中的buffer初始化为0，将count初始化为1；如果该指针不为NULL，则只将count加1；
  • 在release中，将count减1；如果count为0，则释放open中申请的空间，并把指针置为NULL；
  • 在read里面对该buffer进行读取；
  • 在write里面对该buffer进行赋值（任意赋值即可）
  • 写测试程序进行测试，可以一个进程读，一个进程写，实现进程间通信

Linux字符设备驱动

字符设备是3大类设备（字符设备、块设备、网络设备）中较简单的一类设备、其驱动程序中完成的主要工作是初始化、添加和删除struct cdev结构体，申请和释放设备号，以及填充struct file_operations结构体中断的操作函数，实现struct file_operations结构体中的read()、write()和ioctl()等函数是驱动设计的主体工作。

如图，在Linux内核代码中：

  • 使用struct cdev结构体来抽象一个字符设备；
  • 通过一个dev_t类型的设备号确定字符设备唯一性；
  • 通过struct file_operations类型的操作方法集来定义字符设备提供个VFS的接口函数。

运行结果

先依次执行以下命令：

  1. sudo mknod /dev/demo_drv c 98 0新建结点；
  2. sudo chmod a=rwx /dev/demo_drv给权限；
  3. make编译模块；
  4. sudo insmod demo_drv.ko插入模块；
  5. gcc app1.c -o app1和gcc app2.c -o app2编译测试程序
  6. ./app1和./app2测试进程间通信。
  7. sudo rmmod demo_drv卸载驱动。
